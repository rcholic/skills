# Memory 保留 vs 删除规则

## 必须保留

### 核心信息
- 团队成员当前状态和联系方式
- 未完成任务的进度
- 正在进行中的讨论/会议
- 关键配置（API keys 已用 *** 隐藏）
- 项目里程碑和deadline

### 团队协作
- 会议决策和投票结果
- 任务分配（谁负责什么）
- 待处理的事项清单

## 可以删除

### 过时信息
- 已完成超过1周的任务细节
- 已过期/取消的任务记录
- 超过2周的会话历史

### 冗余内容
- 完整的代码片段（代码在仓库里）
- 已过时的临时文件
- 测试/实验性质的记录

## 存档规则

- **archive/** 目录：存放超过7天的 memory 文件
- 格式：`{TIMESTAMP}_{原文件名}`
- 不自动删除，只存档

## 决策流程

1. 调用 session_status 查看 context 使用率
2. 如果 > 75%：
   - 搜索 memory/ 中的过时内容
   - 移动到 archive/
   - 更新 MEMORY.md 保留核心信息
3. 汇报存档结果和剩余使用率
